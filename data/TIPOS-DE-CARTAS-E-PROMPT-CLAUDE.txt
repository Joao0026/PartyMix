================================================================================
1. MODO AMIGOS — CARTAS NORMAIS (categories)
================================================================================
Ficheiro: data/friends/[pack].json
Jogo: mapa + desafios, 1 telemóvel que passa na mesa.

--------------------------------------------------------------------------------
1.1 telepatia
--------------------------------------------------------------------------------
O que faz: dá um TEMA; dois jogadores dizem uma palavra ao mesmo tempo.
Se coincidir → ganham / escolhem penalização.

Campos obrigatórios:
  text  (string) — inclui o tema E a instrução do jogo

Exemplo CORRETO:
{
  "text": "Tema: coisas que se dizem no Uber. Dois jogadores dizem uma palavra ao mesmo tempo."
}

Exemplo ERRADO (só tema, falta instrução):
  "Tema: comida favorita às 3 da manhã"

Variações úteis no text: "... Palavra ao mesmo tempo." | "... Sincronia."

--------------------------------------------------------------------------------
1.2 perguntas (Sabichão)
--------------------------------------------------------------------------------
O que faz: quiz com 4 opções; errar = penalização (goles se ativado).

Campos obrigatórios:
  text        (string) — pergunta
  answer      (string) — resposta certa (tem de existir em choices)
  choices     (array)  — exatamente 4 strings
  difficulty  (string) — "facil" | "medio" | "dificil"

Exemplo:
{
  "text": "Que país ganhou o Euro 2016?",
  "answer": "Portugal",
  "choices": ["França", "Portugal", "Espanha", "Alemanha"],
  "difficulty": "facil"
}

--------------------------------------------------------------------------------
1.3 desenho (Rabiscos)
--------------------------------------------------------------------------------
O que faz: desenhar sem falar; equipa adivinha (~60 s no jogo).

Campos obrigatórios:
  text  (string) — começa por "Desenha ..."

Exemplo:
{
  "text": "Desenha alguém a tentar pedir comida às 3 da manhã."
}

--------------------------------------------------------------------------------
1.4 mimica (Gestos)
--------------------------------------------------------------------------------
O que faz: representar sem falar; equipa adivinha (~45 s).

Campos obrigatórios:
  text  (string) — "Representa ..." ou "Imita ..."

Exemplo:
{
  "text": "Representa alguém a fingir que está sóbrio."
}

Evita 50 cartas todas com "Fingir que estás..." — varia o verbo.

--------------------------------------------------------------------------------
1.5 proibido (Palavra Tabu)
--------------------------------------------------------------------------------
O que faz: palavra secreta; não podes dizer as 5 palavras proibidas.

Campos obrigatórios:
  text             (string) — menciona a palavra alvo e a regra
  forbiddenWords   (array)  — 5 palavras (não uses a palavra alvo aqui)

Exemplo:
{
  "text": "Palavra: Ressaca. Faz a equipa adivinhar sem dizer as palavras proibidas.",
  "forbiddenWords": ["bebida", "dor", "cabeça", "água", "noite"]
}

Formato alternativo no text (também válido):
  "Palavra: IRS. Tabu."

--------------------------------------------------------------------------------
1.6 caos
--------------------------------------------------------------------------------
O que faz: regra temporária para a MESA durante várias rondas.

Campos obrigatórios:
  text                 (string)
  is_ongoing           (boolean) — true
  ongoing_rounds       (number)  — 1 a 4 típico
  ongoing_instruction  (string) — resumo curto da regra ativa

Exemplo:
{
  "text": "Regra temporária: durante 2 rondas, quem disser nomes próprios bebe 1 golo.",
  "is_ongoing": true,
  "ongoing_rounds": 2,
  "ongoing_instruction": "Sem nomes próprios. Quem falhar bebe 1."
}

================================================================================
2. MODO AMIGOS — CARTA ESPECIAL ★
================================================================================

--------------------------------------------------------------------------------
2.1 impostor ★ (mapa — casa roxa 🎭)
--------------------------------------------------------------------------------
O que faz: passa o telemóvel a todos; 1 jogador vê pergunta ERRADA; grupo discute.
  Acertam quem era → impostor bebe
  Falham → impostor distribui goles

Campos obrigatórios:
  correctQuestion  (string) — pergunta para quase todos
  wrongQuestion    (string) — pergunta diferente / mais embaraçosa para o impostor

NÃO uses só "text" neste tipo.

Exemplo:
{
  "correctQuestion": "Quem do grupo sobreviveria num apocalipse?",
  "wrongQuestion": "Quem é o mais medricas?"
}

Tom: perguntas curtas, respondíveis em voz alta, sem ser ofensivo demais.

3. MODO FAMÍLIA
================================================================================
Ficheiro: data/family/[pack].json
Mesmos formatos que Amigos para: telepatia, perguntas, desenho, mimica, proibido.

Diferenças:
  - Tom mais leve; evita álcool, sexualidade, humilhação.
  - Setup padrão sem categoria caos (podes ter no JSON mas não é o foco).
  - impostor: se usares, perguntas suaves (evita "quem mentiu", "quem é falso").

Exemplo telepatia família:
{
  "text": "Tema: sobremesa favorita. Dois jogadores dizem uma palavra ao mesmo tempo."
}

4. MODO CASAL — CARTAS NORMAIS (categories)
================================================================================
Ficheiro: data/couple/[pack].json
2 jogadores, 1 telemóvel.

--------------------------------------------------------------------------------
4.1 romantico
--------------------------------------------------------------------------------
Campos: text

{ "text": "Diz uma coisa pequena que o teu parceiro faz e que te deixa feliz." }

--------------------------------------------------------------------------------
4.2 picante
--------------------------------------------------------------------------------
Campos: text — sensual mas com consentimento; nada explícito/gráfico.

{ "text": "Sussurra uma vontade que gostavas de experimentar (se ambos quiserem)." }

--------------------------------------------------------------------------------
4.3 verdade
--------------------------------------------------------------------------------
Campos: text — pergunta honesta.

{ "text": "Qual foi o momento em que te sentiste mais próximo do teu parceiro?" }

--------------------------------------------------------------------------------
4.4 acao
--------------------------------------------------------------------------------
Campos:
  text         (string)
  time_limit   (number, opcional) — segundos, ex.: 60

{
  "text": "Dá uma massagem nas mãos durante 1 minuto.",
  "time_limit": 60
}

--------------------------------------------------------------------------------
4.5 roleplay
--------------------------------------------------------------------------------
Campos: text — começa por "Cena: ..."

{ "text": "Cena: dois desconhecidos encontram-se num bar." }

--------------------------------------------------------------------------------
4.6 quiz / casal_pergunta
--------------------------------------------------------------------------------
Pergunta sobre o parceiro; a resposta certa é sempre dita por ele/ela.

Campos:
  text    (string)
  answer  (string) — usa "Resposta do parceiro"

{
  "text": "Qual é o prato favorito do teu parceiro?",
  "answer": "Resposta do parceiro"
}

5. MODO BEBER — ESTRUTURA DO FICHEIRO
================================================================================
Ficheiro: data/drink/decks.json

{
  "name": "Beber - Baralhos",
  "mode": "drink",
  "pack": "base",
  "description": "...",
  "decks": {
    "waterfall": {
      "label": "🌊 Waterfall & Beber",
      "desc": "Quem bebe, bebe muito",
      "premium": false,
      "cards": [ ... ]
    },
    "especiais": {
      "label": "🛡️ Especiais",
      "desc": "Agentes, alianças, espelhos e mini bosses",
      "premium": false,
      "cards": [ ... ]
    }
  }
}

Cada carta dentro de "cards" tem sempre:
  type   (string) — ver tipos abaixo
  emoji  (string) — um emoji
  title  (string) — título curto no ecrã

Mais campos conforme o type (ver secções 5.1 e 5.2).
Texto com "bebe X goles" / "distribui X" → contador de goles automático.

================================================================================
5.1 MODO BEBER — CARTAS NORMAIS (type)
================================================================================

--------------------------------------------------------------------------------
type: beber
--------------------------------------------------------------------------------
O que faz: indica quem bebe quantos goles.

Campos obrigatórios: type, emoji, title, text

{
  "type": "beber",
  "emoji": "🍺",
  "title": "Bebe 2!",
  "text": "O jogador com o cabelo mais comprido bebe 2 goles."
}

--------------------------------------------------------------------------------
type: desafio
--------------------------------------------------------------------------------
O que faz: Eu Nunca, verdade, desafio de grupo. Pode ou não incluir goles no text.

{
  "type": "desafio",
  "emoji": "🙅",
  "title": "Eu Nunca",
  "text": "EU NUNCA enviei mensagem ao ex bêbado. Quem já fez bebe 1!"
}

--------------------------------------------------------------------------------
type: regra
--------------------------------------------------------------------------------
O que faz: nova regra na mesa até sair outra regra ou fim de jogo.

{
  "type": "regra",
  "emoji": "📜",
  "title": "Nova Regra!",
  "text": "Quem disser \"sim\" ou \"não\" bebe 1 gole até ao fim do jogo!"
}

--------------------------------------------------------------------------------
type: poder
--------------------------------------------------------------------------------
O que faz: vantagem — distribuir goles, imunidade, escolher quem bebe.

{
  "type": "poder",
  "emoji": "👑",
  "title": "Poder!",
  "text": "Distribui 3 goles como quiseres."
}

--------------------------------------------------------------------------------
type: sorte
--------------------------------------------------------------------------------
O que faz: imunidade ou anular penalização.

{
  "type": "sorte",
  "emoji": "🍀",
  "title": "Sorte!",
  "text": "Ignoras a próxima carta de beber."
}

--------------------------------------------------------------------------------
type: azar
--------------------------------------------------------------------------------
O que faz: penalização extra.

{
  "type": "azar",
  "emoji": "💀",
  "title": "Azar!",
  "text": "Bebes o dobro da próxima penalização."
}

--------------------------------------------------------------------------------
type: caos
--------------------------------------------------------------------------------
O que faz: regra caótica para TODA a mesa (não confundir com caos do mapa Amigos).

{
  "type": "caos",
  "emoji": "💥",
  "title": "Caos!",
  "text": "Durante 5 minutos, quem fizer perguntas bebe. Responder com pergunta também conta."
}

================================================================================
5.2 MODO BEBER — CARTAS ESPECIAIS ★ (baralho "especiais")
================================================================================

--------------------------------------------------------------------------------
type: agent ★ AGENTE SECRETO
--------------------------------------------------------------------------------
O que faz: UM jogador vê missão secreta; lê em voz alta só a parte pública.
Botões no ecrã:
  Consegui → distribui 3 goles
  Falhei → bebe 2 goles
  Fui apanhado → bebe 4 goles

Campos obrigatórios:
  type, emoji, title, publicText, secretMission

NÃO uses só "text" para agent.

{
  "type": "agent",
  "emoji": "🕵️",
  "title": "Agente Secreto",
  "publicText": "Missão pública: pede ao grupo para contar uma história embaraçosa de infância.",
  "secretMission": "Missão secreta: faz alguém dizer a palavra 'comboio'. Se conseguires, distribuis 3 goles."
}

publicText  = o que dizes à mesa em voz alta
secretMission = objetivo escondido (palavra, gesto, frase que alguém tem de dizer/fazer)

--------------------------------------------------------------------------------
type: alliance ★ ALIANÇA
--------------------------------------------------------------------------------
O que faz: ligas-te a outro jogador (ex.: à esquerda) por 2 rondas; penalizações partilhadas.

Campos obrigatórios: type, emoji, title, text

{
  "type": "alliance",
  "emoji": "🤝",
  "title": "Aliança!",
  "text": "Escolhe o jogador à tua esquerda. Durante 2 rondas: se um beber, o outro bebe 1; se um falhar desafio, sofre junto."
}

--------------------------------------------------------------------------------
type: mirror ★ ESPELHO
--------------------------------------------------------------------------------
O que faz: escudo — imune à próxima carta de beber OU reflete penalização para outro.

{
  "type": "mirror",
  "emoji": "🛡️",
  "title": "Espelho!",
  "text": "Ganhas um escudo. Ficas imune à próxima carta de beber e podes passar essa penalização para outro jogador."
}

--------------------------------------------------------------------------------
type: miniboss ★ MINI BOSS
--------------------------------------------------------------------------------
O que faz: desafio para A MESA INTEIRA com tempo e critério claro.
  Grupo ganhou → todos distribuem 2 goles
  Grupo falhou → todos bebem 2 goles

O text deve dizer: O QUÊ, QUANTOS, EM QUANTOS SEGUNDOS, e consequência.

{
  "type": "miniboss",
  "emoji": "👹",
  "title": "Mini Boss!",
  "text": "Todos têm 20 segundos para nomear 10 marcas portuguesas. Se conseguirem, todos distribuem 2. Se falharem, todos bebem 2."
}

--------------------------------------------------------------------------------
type: impostor ★ (Modo Beber)
--------------------------------------------------------------------------------
Igual ao mapa: passa telemóvel; 1 vê wrongQuestion, resto vê correctQuestion.
  Mesa descobre → impostor bebe 3
  Mesa falha → impostor distribui 3

Campos obrigatórios:
  type, emoji, title, correctQuestion, wrongQuestion

(Não precisas de "text".)

{
  "type": "impostor",
  "emoji": "🎭",
  "title": "Impostor!",
  "correctQuestion": "Quem seria o melhor DJ da festa?",
  "wrongQuestion": "Quem tem o pior timing para piadas?"
}

6. MODO CARTAS (estilo Cartas Contra Tugas)
================================================================================
Ficheiro: data/cards/[pack].json

{
  "name": "Modo Cartas - Base",
  "pack": "base",
  "mode": "cards",
  "description": "...",
  "white": [ "resposta curta 1", "resposta curta 2", ... ],
  "black": [ "frase com buraco ___.", ... ]
}

--------------------------------------------------------------------------------
white — carta branca (resposta)
--------------------------------------------------------------------------------
O que é: frase CURTA e engraçada que completa a pergunta preta.
Formato: string simples no array (não é objeto).

CORRETO:
  "O grupo da família no WhatsApp"
  "Um gin tónico a 12 euros"
  "A fila do multibanco no dia 25"

ERRADO (isto é carta de poder/beber, NÃO é white):
  "Ignoras 1 regra à tua escolha"
  "Bebe 2 goles sem motivo"

--------------------------------------------------------------------------------
black — carta preta (pergunta)
--------------------------------------------------------------------------------
O que é: frase com buraco ___ ; um jogador (czar) lê; outros respondem com whites.

CORRETO:
  "A festa acabou quando alguém trouxe ___."
  "O Benfica perdeu porque ___."

ERRADO:
  frases sem ___
  cartas de penalização

7. MISTER WHITE (não é JSON em data/)
================================================================================
Pares no código — formato conceptual:

  civil (maioria):     "Pizza"
  undercover (1):      "Focaccia"    ← palavra PARECIDA, não oposta
  Mr. White (1):       sem palavra

ERRADO para undercover: "Sol" / "Lua" (opostos — muito fácil de descobrir)
CORRETO: "Pizza" / "Focaccia", "Café" / "Chá", "Carro" / "Carrinha"

8. O QUE NÃO É CARTA JSON (referência)
================================================================================
Casas do mapa (só com "Casas especiais" no setup): duelo, shot, regra, roubo, proteção,
  bónus, risco, todos, impostor — lógica no MapGame, não em data/.

Mini-jogos no mapa (🎮): maior_menor, grupo, espio, 10_segundos, batalha, sync, password.

Roleta do Modo Beber: segmentos fixos no código (Bebe 1/2/3, Distribui 3, etc.).

Comunidade: jogadores submetem → admin aprova → entra como carta/desafio normal.

10. ERROS COMUNS (evitar ao escrever / ao rever PDFs)
================================================================================
1. Telepatia só com "Tema: ..." sem instrução de jogo.
2. Agent com só "text" — falta publicText + secretMission.
3. Impostor com só "text" — falta correctQuestion + wrongQuestion.
4. Modo Cartas: whites/blacks que são regras de beber ou poderes.
5. Proibido sem forbiddenWords ou com menos de 5 palavras.
6. Perguntas sem 4 choices ou answer que não está nas choices.
7. Miniboss vago ("desafio para a mesa") sem tempo nem critério de vitória/derrota.
8. Mister White: palavras undercover opostas em vez de parecidas.
9. Muitas cartas iguais ("Quem X bebe 1 gole") — reformula ou corta.
10. Conteúdo copiado de outros jogos — usa só como inspiração; texto 100% teu.
